# orchestration.md — MINTORAIN 바이브코딩 시스템 제어 및 조율

> 총주방장 역할: 에이전트 간 충돌 방지, 권한 관리, 에스컬레이션 체계

## 실행 우선순위

```
1순위: 보안 규칙 (CLAUDE.md 필수 규칙)
2순위: 사용자 명시적 지시
3순위: 에이전트 자동 판단
4순위: 기본 동작 (docs/ 참조)
```

## 충돌 해결 규칙

### 파일 수정 충돌
```
상황: 두 에이전트가 같은 파일을 수정하려 할 때
해결: 먼저 시작한 에이전트가 완료될 때까지 대기
원칙: 절대 동시 수정 불가
```

### 의견 충돌
```
상황: planner와 code-reviewer의 의견이 다를 때
해결: 보안 > 성능 > 가독성 > 편의성 순으로 판단
에스컬레이션: 해결 불가 시 사용자에게 선택지 제시
```

## 에스컬레이션 체계

### Level 1: 자동 처리
- 단순 빌드 에러
- 린트 경고
- 타입 에러

### Level 2: 사용자 알림
- 아키텍처 변경이 필요한 경우
- 외부 서비스 설정 변경
- 데이터베이스 마이그레이션

### Level 3: 사용자 확인 필수
- 파일 삭제
- 배포 실행
- 환경변수 변경
- 결제 관련 코드 수정
- RLS 정책 변경

## 컨텍스트 관리

### 자동 컴팩트 타이밍
```
트리거: 컨텍스트 사용량 50% 도달 시
동작: MEMORY.md에 현재 상태 저장 후 컴팩트 제안
보존: 현재 작업 파일 경로, 변수명, 진행 상태
```

### 세션 전환 프로토콜
```
세션 종료 시:
1. 현재 작업 상태를 MEMORY.md에 기록
2. 미완료 작업 목록 저장
3. 중요 결정사항 요약

세션 시작 시:
1. MEMORY.md 첫 200줄 로드
2. 미완료 작업 확인
3. 이전 컨텍스트 복원
```

## 병렬 실행 규칙

### 병렬 가능한 작업
- 서로 다른 파일의 코드 작성
- 독립적인 컴포넌트 생성
- 테스트와 문서 동시 작성

### 순차 실행 필수 작업
- DB 마이그레이션 → 코드 수정 → 테스트
- 인증 설정 → 보호된 라우트 구현
- API 엔드포인트 → 프론트엔드 연동
