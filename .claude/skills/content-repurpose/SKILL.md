# 원소스 멀티유즈 콘텐츠 변환 스킬

## 변환 매트릭스

```
[원본 콘텐츠]
    │
    ├─→ [유튜브 롱폼] (10-15분, 교육적)
    ├─→ [유튜브 쇼츠] (30-60초, 핵심만)
    ├─→ [네이버 블로그] (2000자+, SEO 최적화)
    ├─→ [쓰레드/X] (280자, 훅+인사이트)
    ├─→ [인스타 캐러셀] (5-10장, 비주얼)
    ├─→ [이메일 뉴스레터] (가치+CTA)
    └─→ [카카오 채널 포스트] (친근한 톤)
```

## 플랫폼별 변환 규칙

### 쓰레드 (Threads)
```
글자수: 500자 이내
구조: 훅 1줄 → 본문 3-5줄 → CTA 1줄
톤: 대화체, 이모지 적절히
해시태그: 3-5개 (마지막 줄)
바이럴 패턴: 논쟁적 의견, 실패담, 숫자 데이터
```

### 네이버 블로그
```
글자수: 2000-3000자
구조: 제목(키워드 포함) → 서론 → 본론(소제목 3-5개) → 결론+CTA
이미지: 최소 5장 (텍스트 이미지 포함)
SEO: 키워드 반복 3-5회, 이웃 태그
```

### 인스타그램 캐러셀
```
장수: 7-10장
1장: 표지 (궁금증 유발 제목)
2-8장: 핵심 내용 (슬라이드당 1포인트)
마지막장: CTA (저장/공유/팔로우)
사이즈: 1080x1350 (4:5)
```

### 이메일 뉴스레터
```
제목: 오픈율 높은 패턴 (숫자, 질문, 긴급성)
길이: 500-800자
구조: 인사 → 스토리 → 핵심 가치 → CTA 1개
톤: 1:1 대화, 친근하게
```

## n8n 자동화 워크플로
```
[RSS/웹훅: 새 콘텐츠 감지]
    → [OpenAI: 원본 분석 및 핵심 추출]
    → [Switch: 플랫폼별 분기]
        ├→ [OpenAI: 쓰레드용 변환] → [Threads API 포스팅]
        ├→ [OpenAI: 블로그용 변환] → [네이버 블로그 API]
        ├→ [OpenAI: 뉴스레터 변환] → [ConvertKit 발송]
        └→ [OpenAI: 캐러셀 스크립트] → [이미지 생성] → [인스타 API]
```

## AI 프롬프트 템플릿

### 콘텐츠 변환 시스템 프롬프트
```
당신은 MINTORAIN 바이브코딩의 콘텐츠 매니저입니다.
원본 콘텐츠를 {platform}에 맞게 변환하세요.

브랜드 보이스: 친근하지만 전문적, 실용적, 행동 유도적
대상: 코딩 비전공 1인 창업가 (30-50대)
금지: 기술 전문 용어, 영어 남발, 과한 이모지
필수: 구체적 사례, 실행 가능한 팁, 명확한 CTA
```
