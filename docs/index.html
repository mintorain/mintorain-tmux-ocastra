Quick Start

5분만에 5명의 팀

  1. 1

    사전 도구 설치

    OS에 맞춰 한 줄씩.

    🍎 macOS
    brew install tmux tmuxp
npm i -g @anthropic-ai/claude-code
    🐧 Linux
    sudo apt install tmux python3-pip
pip3 install --user tmuxp
npm i -g @anthropic-ai/claude-code
    🪟 Windows (WSL 불필요)
    winget install arndawg.zellij-windows
winget install Microsoft.PowerShell
npm i -g @anthropic-ai/claude-code
  2. 2

    키트 클론

    git clone https://github.com/mintorain/mintorain-tmux-ocastra
cd mintorain-tmux-ocastra/mintorain-tmux-ocastra-main
  3. 3

    실행 → 10초 안에 5분할 등장

    🍎 macOS
    ./mintorain-team.command
    🐧 Linux / WSL2
    bash ./mintorain-tmuxp.sh
    🪟 Windows
    pwsh ./mintorain-team.ps1

FAQ

자주 묻는 질문

5명의 Claude를 동시에 실행하면 비용이 5배 드나요? ▼

네, 토큰 사용량은 늘어납니다. 단, Claude Code Pro/Team 정액제를 사용하면 한도 내에서 부담 없이 사용 가능합니다. 또한 5개를 항상 동시 사용할 필요는 없으며, 필요한 패인만 활용하면 됩니다.

Windows에서 정말 WSL 없이 동작하나요? ▼

네. tmux 대신 Zellij (Rust 기반 모던 멀티플렉서) 의 Windows 네이티브 빌드를 사용합니다. winget install arndawg.zellij-windows 로 설치하면 PowerShell만으로 100% 같은 5분할 워크플로가 가능합니다.

각 패인이 같은 코드베이스를 본다면, 충돌은 어떻게 막나요? ▼

orchestration.md 가 권한 매트릭스를 정의합니다. planner/code-reviewer 는 읽기만, landing-builder/n8n-architect 는 쓰기 가능, 메인 에이전트는 모든 권한. 또한 두 에이전트가 같은 파일을 동시 수정 못하게 "먼저 시작한 에이전트 우선" 규칙이 있습니다.

역할별 system prompt를 커스터마이징할 수 있나요? ▼

네. mintorain-zellij.kdl (Windows) 또는 tmux 스크립트의 --append-system-prompt 인자를 직접 수정하면 됩니다. .claude/agents/*.md 의 역할 정의도 함께 갱신하세요.

비전공자가 사용해도 되나요? ▼

네. 두온교육 바이브코딩 철학이 "비전공자가 AI 도구로 결과물을 만든다" 입니다. 모든 문서가 한국어로, 비전공자 기준으로 작성되어 있습니다. 막히는 부분은 mintorain.duonedu.net 으로 문의주세요.
